css、js和html学习

iiitoo
2017-05-16
学习, 网站建设

刚刚在看一本JavaScript书的前言的时候，突然想到一种方法。本来在我学习JavaScript和css的时候，混乱充斥着我学习这些的一切，有时到了绝望的时候，认为它们是无解的，根本无法前进。我在学习过程中，不管别的什么，拿起书就看，总是认为这样就已经算是看书，至于效果如何，就看脑子了。但是，问题就在于此，当看着看着，问题太多，然后相互交织在一起根本就没有进本的余地。
今天，突然想到的一种方法。让我一下子好像是学习的一种解决办法，并把他记下来。

认识学习的东西是什么，能解决我什么。（这来以css为理解）

如果要学习css，首先，我要知道它是做什么的？也许这个如果往深里想也就复杂了，我只需要了解他的一个概念就可以了。
然后就是它能解决我的什么问题？解决我什么问题，当然需要什么工具，我们不可能去拿锅做网页设计。当然，他需要css、html和JavaScript等其他工具来做网页。
那么这里css是做什么的？
html是做什么的？
js是做什么？
他们怎么相互关系的，他们有什么重叠的功能，这些重叠的功能到底用什么工具比较方便好用。
他们的主要功能是什么？

给他们的功能分类

我们知道了它们能做的事的时候，我们就给他们一个系统的分类方式，比如css，有控制字体的，有控制布局的，有控制动画的等等。等这些系统建立完备以后，我们所做的或者说所学的只是往这个系统里添加就可以。

###总结
这可能就是一种逻辑的分类法吧。我觉得这样应该会很快的学到自己想要学的东西，也很清楚，自己有什么东西，没有什么东西，我们只是想办法去获取这些缺失的知识就好了。
